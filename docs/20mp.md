# Feature plan: max-megapixel cap for BatchStyler

## Goal

BatchStyler currently opens images with raw `Image.open().convert("RGB")`, so it
always processes at full resolution.  PictureStyler (GUI app) uses
`PhotoManager.load(path, max_megapixels=20.0)` and downscales any image that
exceeds the limit before passing it to the engine.

This plan makes BatchStyler consistent: the same 20 MP cap is applied by
default, with an explicit `--max-megapixels` flag to override or disable it.

class TestMaxMegapixels:

    def test_oversized_image_is_downscaled(self, tmp_path):
        """An image above the cap is loaded at reduced size by cmd_apply_style_chain."""
        # 6000×4000 = 24 MP; cap = 20 → expect ~5477×3651
        big_img = Image.new("RGB", (6000, 4000))
        img_path = tmp_path / "big.jpg"
        big_img.save(img_path, format="JPEG")

        captured: list[Image.Image] = []
        def fake_apply(source, style_id, **kwargs):
            captured.append(source)
            return source.copy()

        # patch engine + catalog so command runs without real models
        with patch("src.batch_styler.commands.StyleTransferEngine") as MockEng, \
             patch("src.batch_styler.commands._catalog.REPO_ROOT", tmp_path):
            instance = MockEng.return_value
            instance.apply.side_effect = fake_apply
            instance.load_model.return_value = None
            instance.unload_model.return_value = None

            chain_yml = _write_single_step_chain(tmp_path)
            _write_style_catalog(tmp_path)

            bs_commands.cmd_apply_style_chain(
                img_path, chain_yml,
                tile_size=256, overlap=32, use_float16=False,
                max_megapixels=20.0,
            )

        assert len(captured) == 1
        w, h = captured[0].size
        assert w * h / 1_000_000 <= 20.0 + 0.01     # within rounding tolerance
        assert w < 6000 and h < 4000                 # was actually downscaled

    def test_cli_max_megapixels_flag_passed_through(self, tmp_path, monkeypatch):

### Private helpers needed in the test module

Add these near the top of the `TestMaxMegapixels` section (or alongside the

```python
def _write_single_step_chain(tmp_path: Path) -> Path:
    """Write a minimal single-step style-chain YAML and return its path."""
    chain_dir = tmp_path / "chains"
    chain_dir.mkdir(exist_ok=True)
    yml = chain_dir / "chain.yml"
    yml.write_text(
        "version: 1\nsteps:\n- style: TestStyle\n  strength: 100\n",
        encoding="utf-8",
    )
    return yml


def _write_style_catalog(tmp_path: Path) -> None:
    """Write a minimal styles catalog with a 'TestStyle' entry."""
    styles_dir = tmp_path / "styles"
    styles_dir.mkdir(exist_ok=True)
    model = styles_dir / "teststyle.onnx"
    model.write_bytes(b"")          # placeholder so path exists
    catalog = styles_dir / "catalog.json"
    catalog.write_text(
        json.dumps({"styles": [{"id": "teststyle", "name": "TestStyle",
                                "model_path": "styles/teststyle.onnx",
                                "preview_path": ""}]}),
        encoding="utf-8",
    )
```

## Scope not included

| Item | Reason |
|------|--------|
| `--style-overview` PDF commands | Cap affects only per-image quality, PDF layout is unchanged. Tests for those commands can reuse `_write_style_catalog` and follow the same patch pattern if desired. |
| Settings-file auto-detection | Reading `~/.picture_styler/settings.json` in batch mode would add coupling and surprises in scripting contexts. Explicit CLI flag is clearer. |
